01 · Product Overview

产品概述

轻量化山岳信息查询工具，专注收录国内名山，为旅游规划、地理学习、户外参考提供直观的数据查询服务。

🏔
数据查询
收录 186 座国内名山，含地理坐标、海拔、山系、省市归属、景区等级等结构化数据
🌤
实时天气
接入和风天气 API，展示各山岳当前天气及未来 15 天预报，支持温度、风力、湿度查看
🏷
分类筛选
按五岳、四大佛山、世界遗产、5A景区、雪山冰川、道教名山等标签快速过滤，支持海拔排序
📌
打卡足迹
标记已到访山岳，记录打卡时间，统计足迹省份数及各分类完成进度，支持分享
❤️
收藏夹
一键收藏感兴趣的山岳，独立收藏夹页面集中管理，数据存于云端跨设备同步
🔍
搜索
支持山岳名称、省市地区关键词搜索，实时过滤，快速定位目标山岳

04 · Feature Modules

功能模块说明

各功能的交互逻辑、数据来源与技术实现方案。

功能 页面 交互说明 数据来源 实现
山岳列表 首页 双列卡片网格，每张卡片展示名称、省市、海拔、实时天气、打卡/收藏状态 本地 JSON 数据 本地
分类筛选 首页 横滑标签芯片，点击过滤列表；支持"已打卡"快捷筛选；右侧按钮切换海拔/默认排序 tags 字段匹配 本地
关键词搜索 搜索页 输入山名或省市关键词，实时过滤，支持拼音首字母搜索 本地过滤 本地
实时天气 首页卡片、详情页 首页懒加载（分批 5 条），详情页进入时触发；展示温度、天气描述、风力、湿度、能见度 和风天气 /weather/now API
15天天气预报 详情页 横滑卡片条，今日高亮，展示日期/图标/最高温，可展开查看详细参数 和风天气 /weather/15d API
地理信息 详情页 四格信息卡：所属山系、主峰海拔、地理坐标、景区等级；标签行展示所有分类标签 本地 JSON 本地
收藏 首页、详情页、收藏夹 点击爱心图标切换收藏状态，数据存储于云数据库 favorites 集合，按 openid 隔离 云数据库 云DB
打卡记录 详情页、我的 点击"标记打卡"按钮写入 checkins 集合，记录 mountain_id + 时间戳；可选 GPS 围栏验证 云数据库 云DB
打卡进度统计 我的 按分类展示进度条（已打卡/总数），统计足迹省份数、覆盖山系数 云数据库聚合 云DB
收藏夹管理 收藏标签页 展示所有收藏山岳，支持长按删除，按收藏时间降序排列 云数据库 云DB

云函数：getWeather（天气代理）
// cloudfunctions/getWeather/index.js // 统一代理天气请求，避免 API Key 暴露在前端 const cloud = require('wx-server-sdk') const got = require('got') cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV }) exports.main = async (event) => { const { lat, lng, type = 'now' } = event const KEY = process.env.QWEATHER_KEY const path = type === '15d' ? 'weather/15d' : 'weather/now' const url = `https://devapi.qweather.com/v7/${path}` const { body } = await got(url, { searchParams: { location: `${lng},${lat}`, key: KEY, lang: 'zh', unit: 'm' }, responseType: 'json' }) return body }
06 · Data Schema

数据结构

山岳基础数据以 JSON 本地存储，用户数据（收藏、打卡）存于云数据库。

山岳数据（本地 JSON）
{ "id": "taishan", // 唯一标识 "name": "泰山", "province": "山东", "city": "泰安市", "altitude": 1545, // 海拔，单位 m "coords": { "lat": 36.2544, "lng": 117.1006 }, "mountain_system": "泰山山脉", "tags": ["五岳", "世界遗产", "5A景区"], "level": "5A", // 景区等级 "heritage": true, // 是否世界遗产 "intro": "泰山，古称岱宗…" }
云数据库：favorites（收藏）
{ "_id": "auto", "_openid": "用户openid（自动）", "mountain_id": "taishan", "created_at": "2024-03-15T10:30:00Z" }
云数据库：checkins（打卡）
{ "_id": "auto", "_openid": "用户openid（自动）", "mountain_id": "taishan", "mountain_name": "泰山", // 冗余字段，便于列表展示 "province": "山东", "checkin_at": "2024-03-15T10:30:00Z", "location": { // 可选，GPS 验证 "lat": 36.25, "lng": 117.10 } }
07 · File Structure

项目文件结构

微信云开发标准目录结构，前后端分离，工具函数统一在 utils/ 下管理。

mountain-app/
miniprogram/ ← 前端代码
data/
mountains.js ← 186 座山岳基础数据
utils/
weather.js ← 天气 API 封装
storage.js ← 本地缓存工具（收藏/打卡）
pages/
index/ ← 首页（列表+筛选）
detail/ ← 详情页（天气+地理+打卡）
search/ ← 搜索页
favorites/ ← 收藏夹
profile/ ← 我的（统计+打卡记录）
app.js ← 全局初始化，wx.cloud.init()
app.json ← 页面路由 + 标签栏配置
app.wxss ← 全局公共样式
cloudfunctions/ ← 云函数代码
getWeather/ ← 天气代理（保护 API Key）
getUserStats/ ← 聚合打卡统计
toggleFavorite/ ← 收藏 增/删
checkin/ ← 打卡写入 + GPS 验证
project.config.json ← 小程序 AppID + 云环境配置
